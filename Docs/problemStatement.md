# Mutual Fund FAQ Assistant (Facts-Only Q&A)

> [!IMPORTANT]
> **Core Mandate:** This system is a strictly facts-only FAQ assistant. It must prioritize compliance, accuracy, and verifiability over intelligence. It is forbidden from offering investment advice, recommendations, opinions, or performance comparisons.

---

## 📌 Project Overview
The **Mutual Fund FAQ Assistant** is a lightweight, Retrieval-Augmented Generation (RAG)-based solution designed within the context of the **Groww** product ecosystem. Its primary goal is to resolve objective, factual, and verifiable queries about mutual fund schemes using official, public financial documents.

---

## 🎯 Objectives
- Build a RAG system that answers factual queries about mutual fund schemes.
- Retrieve information exclusively from a curated corpus of official public sources (AMCs, AMFI, SEBI).
- Deliver concise, source-backed responses with strict output formats.
- Politely but firmly refuse any advisory or non-factual inquiries.

---

## 👥 Target Users
1. **Retail Investors:** Users seeking to compare mutual fund details (such as exit loads, expense ratios, etc.) without marketing bias or unsolicited advice.
2. **Support & Content Teams:** Teams who handle high-volume, repetitive, factual queries about mutual funds.

### 2. FAQ Assistant Requirements
The assistant must answer objective queries such as:
- Expense ratios of schemes
- Exit load percentages and timelines
- Minimum SIP/Lump sum investment amounts
- ELSS lock-in periods
- Riskometer classifications
- Benchmark indices
- Fund management and fund manager details (e.g., name, tenure, background)
- Instructions for downloading capital gains and holding statements

#### ⚙️ Response Constraints
Every answer generated by the assistant must adhere to this strict structure:
- **Length Limit:** Maximum of 3 sentences.
- **Citation:** Include exactly one official source/citation link.
- **Footer:** Add a footer in the exact format: `Last updated from sources: <date>`
- **No Inference:** The assistant must not infer, calculate, estimate, or generate conclusions from source data and may only provide information explicitly stated in official sources.
- **Latest Documents:** When multiple versions of a document exist, the latest available version must be used.
- **Fallback Response:** If a verified answer cannot be found in the approved corpus, the assistant should return a fallback response indicating that the information could not be located in the approved sources.

### 3. Refusal Handling
The assistant must detect and decline all queries requesting investment advisory, opinion, or performance speculation (e.g., *"Should I buy fund X?"* or *"Which fund has better returns?"*). 

- **Response Tone:** Polite, clear, and neutral.
- **Content:** Reiterate the facts-only constraint.
- **Redirection:** Provide a helpful, educational link to official resources (e.g., SEBI Investor Education or AMFI site).

### 4. Minimal User Interface
Provide a simple, elegant UI featuring:
- A welcome message
- Three predefined example questions
- A prominent disclaimer snippet:
  > **"Facts-only. No investment advice."**

## 🛡️ Critical Constraints & Compliance

| Domain | Policy & Requirements |
| :--- | :--- |
| **Data & Sources** | Use **only** official public sources (AMC, AMFI, SEBI). Third-party blogs, news aggregators, or forums are strictly prohibited. |
| **Privacy & Security** | Do **not** collect, store, or process sensitive personal data (e.g., PAN, Aadhaar, bank accounts, OTPs, email addresses, phone numbers). |
